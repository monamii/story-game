# AI動作確認セットアップ手順書

目的: コード修正のたびに作者が手作業でやっている動作確認（起動→移動→会話→選択肢）を、
Claude Code に任せられるようにする。テスト基盤の導入ではなく、
「AIが代わりに手動確認できる窓口」を最小限用意するだけ（DEV_ENV_PLAN の方針と矛盾しない）。

作業は3ステップ、合計10分程度。このファイルを読んだ Claude Code セッションは、
以下を順に実施してください。

---

## Step 1: `.claude/launch.json` を作る（AIがサーバを自分で起動できるようにする）

プロジェクト直下に `.claude/launch.json` を新規作成:

```json
{
  "version": "0.0.1",
  "configurations": [
    {
      "name": "story-game",
      "runtimeExecutable": "npx",
      "runtimeArgs": ["--yes", "serve", "-l", "8384", "."],
      "port": 8384
    }
  ]
}
```

- これは Claude Code の Preview 機能（AIがブラウザでページを開いて確認する機能）が
  サーバの起動方法を知るための設定ファイル。
- 作者が普段使う Live Server とは別物で、共存できる（ポートも別）。
- npm のインストールは不要（`npx --yes serve` が都度取得して実行する）。

## Step 2: `src/main.js` にデバッグ用フックを1行足す

現在:

```js
new Game(document.getElementById("game")).start();
```

変更後:

```js
const game = new Game(document.getElementById("game"));
game.start();
window.game = game; // 開発用フック: AIやブラウザconsoleから内部状態を確認するため
```

- canvas ゲームは画面上の文字や位置を DOM として読めないため、AIが検証するには
  「内部状態を直接読む窓口」が必要。これがその窓口。
- ゲームの動作には一切影響しない。公開時もそのままで実害はない（気になるなら公開前に削除）。

## Step 3: 動作確認シナリオを CLAUDE.md に追記する

作者がいつも手作業でやっている確認手順を、**作者に聞き取って**文章化し、
CLAUDE.md に「動作確認シナリオ」セクションとして追記する。書式の例:

```markdown
## 動作確認シナリオ（コード修正後にAIが実施する）

1. ページを開き、console にエラーが出ていないことを確認する
2. 浜辺マップでキャラが表示されていることをスクリーンショットで確認する
3. 矢印キーで移動できること（座標が変わること）を確認する
4. NPC に近づいて決定キーで会話が始まることを確認する
5. 会話を最後まで送り、選択肢が2つ表示されることを確認する
6. 選択肢を選ぶと対応する分岐の台詞が出て、フラグが立つことを確認する
```

※ 上記はあくまで例。**実際の手順は作者本人に確認して、現状のゲーム内容に合わせて書くこと。**
シナリオが CLAUDE.md にあれば、以後どのセッションでも「動作確認して」の一言で同じチェックが再現される。

---

## セットアップ後の使い方（作者向けメモ）

- 修正を頼んだあとに「**動作確認もして**」と言えば、AIがサーバを起動し、
  console エラー確認・キー入力のシミュレート・`window.game` 経由の状態検証・
  スクリーンショットでの見た目確認までを行う。
- コミット前にまとめて確認したいときは「**/verify して**」でもよい。
- AIの確認方法は主に「内部状態を読む」＋「スクショを見る」の二本立て。
  演出の細かい気持ちよさ（テンポ・アニメの滑らかさ）は引き続き人間の目で見るのが確実。

## 完了したら

- `.claude/launch.json`・`main.js` の変更・CLAUDE.md の追記をコミットする
